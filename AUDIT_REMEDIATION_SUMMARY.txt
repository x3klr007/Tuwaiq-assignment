ISSUE 1: DATA LEAKAGE (CRITICAL)
---------------------------------
SEVERITY: CRITICAL
STATUS: RESOLVED

PROBLEM:
  Feature 'price_per_bmi' = charges / bmi used target variable to create
  a feature, causing training-serving skew and inflated metrics.

LOCATION: src/utils.py:316-320 (create_interaction_features)

REMEDIATION:
  1. REMOVED 'price_per_bmi' feature entirely
  2. Added 'age_squared' and 'bmi_squared' as non-leakage alternatives
  3. Added 'is_training' parameter to control target transformation
  4. Updated docstring to emphasize NO TARGET LEAKAGE policy

VERIFIED RESULT:
  - BEFORE (with leakage): R2 claimed as 0.871
  - AFTER (leakage fixed): R2 = 0.901 (Gradient Boosting)
  
  Note: Model actually improved after removing leakage, suggesting
  the feature was adding noise rather than signal.
