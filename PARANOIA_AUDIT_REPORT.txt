                    AUDIT TARGET 2: THE DUMMY VARIABLE TRAP
================================================================================

TARGET DESCRIPTION:
Examine if One-Hot Encoding uses drop_first=False, creating perfect 
multicollinearity (dummy variable trap) which damages Linear Regression 
and Ridge model coefficients.

SCAN RESULT:
[!!] CRITICAL: drop_first=False detected in One-Hot Encoding

VERIFICATION:
  File: src/utils.py
  Line: 267 (ORIGINAL)
  
  Code Pattern Found (BEFORE FIX):
    region_dummies = pd.get_dummies(
        df_encoded['region'], 
        prefix='region', 
        drop_first=False  # <-- SILENT KILLER
    )

IMPACT ANALYSIS:
The 'region' variable has 4 categories (northeast, northwest, southeast, 
southwest). With drop_first=False, this creates 4 dummy columns:
  - region_northeast
  - region_northwest
  - region_southeast
  - region_southwest

The sum of these 4 columns equals 1.0 for every row (constant), creating
perfect multicollinearity with the intercept term. This causes:
  1. Singular matrix warnings in Linear Regression
  2. Unstable coefficient estimates
  3. Inflated standard errors
  4. Misleading feature importance

================================================================================
                    REMEDIATION APPLIED
================================================================================

FILE: src/utils.py
LINE: 267

BEFORE (BROKEN):
    region_dummies = pd.get_dummies(df_encoded['region'], prefix='region', drop_first=False)

AFTER (FIXED):
    region_dummies = pd.get_dummies(df_encoded['region'], prefix='region', drop_first=True)

VERIFICATION OF FIX:
  Region dummy columns (3): ['region_northwest', 'region_southeast', 'region_southwest']
  Drop first applied: True (expected 3 for 4 regions)
  Sum of region dummies per row - min: 0, max: 1
  Values are 0 or 1 (not constant 1.0): PASS

Linear Regression coefficients are now stable and interpretable.

STATUS: [PASS] - ISSUE RESOLVED
